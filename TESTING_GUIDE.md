# Table Column Resizer - 预览模式测试指南

## 🎯 测试目标
验证表格列宽调整插件在预览模式下的基本功能是否正常。

## 📋 测试前准备

### 1. 安装插件到Obsidian
1. 打开你的Obsidian库
2. 进入设置 → 社区插件 → 关闭安全模式
3. 点击"浏览"旁边的文件夹图标，打开插件文件夹
4. 创建新文件夹 `table-column-resizer`
5. 复制以下文件到该文件夹：
   - `main.js` (构建生成的)
   - `manifest.json` 
   - `styles.css`
6. 在Obsidian中启用插件

### 2. 准备测试文档
1. 将 `test-table-resizer.md` 文件复制到你的Obsidian库中
2. 打开该文档并切换到**预览模式**（Ctrl/Cmd + E）

## 🧪 测试步骤

### 阶段1：基础功能测试

#### 测试1.1：拖拽手柄显示
1. 将鼠标移动到任意表格列的右边缘
2. **预期结果**：
   - 鼠标光标变为 `col-resize`（左右箭头）
   - 出现半透明的拖拽手柄（6px宽，颜色为主题强调色）
   - 手柄有悬停动画效果

#### 测试1.2：列宽调整
1. 按住鼠标左键拖拽表格列边缘
2. 左右移动鼠标调整列宽
3. **预期结果**：
   - 整列宽度实时跟随鼠标移动
   - 拖拽时表格有蓝色边框高亮
   - 列宽调整有平滑的动画效果

#### 测试1.3：列宽限制
1. 尝试将列宽拖拽到很窄（<50px）
2. 尝试将列宽拖拽到很宽（>500px）
3. **预期结果**：
   - 列宽不会小于50px（最小限制）
   - 列宽不会大于500px（最大限制）

### 阶段2：多表格测试

#### 测试2.1：独立调整
1. 测试文档中的第一个表格
2. 测试文档中的第二个表格
3. **预期结果**：每个表格可以独立调整，互不影响

#### 测试2.2：不同内容表格
1. 测试简单表格（姓名/年龄/城市）
2. 测试长内容表格（产品描述）
3. **预期结果**：所有类型的表格都支持列宽调整

### 阶段3：设置测试

#### 测试3.1：修改限制
1. 进入插件设置
2. 修改最小列宽为100px，最大列宽为300px
3. 重新测试列宽调整
4. **预期结果**：新的限制生效

#### 测试3.2：开关功能
1. 在设置中关闭"在预览模式下启用"
2. 返回文档测试
3. **预期结果**：拖拽功能被禁用
4. 重新开启设置
5. **预期结果**：功能恢复

### 阶段4：用户体验测试

#### 测试4.1：视觉反馈
1. 拖拽时观察视觉变化
2. **预期结果**：
   - 拖拽手柄在悬停时放大
   - 拖拽时表格有蓝色边框
   - 拖拽过程中鼠标光标保持一致

#### 测试4.2：防误触
1. 尝试在表格内容区域点击和选择文本
2. **预期结果**：正常编辑功能不受影响

## 🐛 常见问题排查

### 问题1：拖拽手柄不显示
**可能原因**：
- CSS文件未正确加载
- 插件未启用
- 不在预览模式

**解决方案**：
1. 检查 `styles.css` 是否在插件文件夹中
2. 确认插件已启用
3. 确保在预览模式下（不是编辑模式）

### 问题2：列宽无法调整
**可能原因**：
- JavaScript错误
- 表格渲染未完成

**解决方案**：
1. 打开开发者工具（Ctrl/Cmd + Shift + I）查看控制台错误
2. 刷新文档或重启Obsidian

### 问题3：设置不生效
**可能原因**：
- 设置未保存
- 需要重新加载

**解决方案**：
1. 修改设置后点击其他地方让设置自动保存
2. 重新打开文档测试
